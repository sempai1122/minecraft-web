System Design Snapshot

  • WebSocket transport via Socket.io for low-latency bi-directional events.
  • REST APIs for auth, profile, history retrieval, and chat bootstrap.
  • Message IDs + server sequence numbers prevent dupes and preserve order.
  • Read receipts + typing events scoped to room participants.
  • File attachments stored as metadata + base64 payload (demo; swap cloud storage in production).

Includes direct messages, optional group chat, timestamps, delivery/read status, presence, and responsive UI.
